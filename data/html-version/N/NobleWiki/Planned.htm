Not sure when these will come in, but definitely on the TODO list;

New Commands PlannedDescription
{{history|events}}Create a timeline using the Timeline javascript widget
{{latex|events}}Create LaTex  markup for support of scientific notation
{{syntax|language}}Switch between the various markup languages. Support Markdown, Creole.
{{section|WikiPage|header}}Non-recursively transclude the section text for any page of the identified TOC header
{{css|media|stylesheet}}Added at top of an editor's home-page to alter CSS used before rendering any wiki page.
{{feedback|template}}Use a template [ see above ] to capture feedback or voting information
{{columns|color}}Sort of like the version use on en.wikipedia.org; specify that content following appears in one, two, or three columns.
{{graph|type|dimension}}Add support for javascript-generated graphs. Bar graphs, line graphs, etc.
{{data|id}}Add dataset storage in two phases. Phase I is JSON creation for reference by SWFs, and JavaScript libraries. Phase II is a read/write API possibly via XmlHttpSocket for AJAX or SWF.
{{blog|entryid}}This would be a new GUI that writes to a dataset page under PageId/Blogentryid and support RSS 1.0. Features are editing, archive, publishing, and a WYSIWYG.
{{form|formid}}Create a simple configuration-driven form generator that would write to a dataset page. Simple form templates should be made available such as voting + graph-view, feedback/mailer, calculation, and dataset data-entry + graph-view.
{{whiteboard|shared}}Create sketches using Flash, and store the output as a bitmap on the server, and the curves as JSON elements.
{{widget|type}}Wdigets will be pre-defined as right-hand elements for a HomePage¬ view; calendar, blog roll, rss feeds, tag cloud, favorites, etc.
{{embed|typeThis is a generalized way to embed applets for java, flash, Quicktime, Windows Media Player, etc.
{{category|names}}Using a category tag should do two things; create an entry in the appropriate category page, and create a list of pages in place.
